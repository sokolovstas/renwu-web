# profile

## Назначение
Микрофронтенд пользовательского профиля: настройки пользователя, персональные данные и уведомления.

## Область ответственности
- Экраны профиля и редактирования пользовательских параметров.
- Интеграция с сервисами пользователя и уведомлений.
- Подключение в shell как remote-приложение.
- Локализация и валидация пользовательских форм.

## Что улучшить
- Добавить security-oriented тесты на изменение профиля.
- Разделить API DTO и view-модели через адаптеры.
- Ввести предсказуемую стратегию optimistic update для настроек.

## Техдолг
- `P0`: усилить контроль прав и валидацию критичных изменений профиля.
- `P1`: унифицировать маппинг API моделей к UI-моделям.
- `P2`: добавить аудит-трейл и интеграционные тесты кросс-сценариев.
