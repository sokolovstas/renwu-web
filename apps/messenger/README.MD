# messenger

## Назначение
Микрофронтенд коммуникаций для переписок, тредов и пользовательских уведомлений.

## Область ответственности
- Рендер экранов сообщений и навигация по тредам.
- Интеграция с `libs/messaging` и кросс-доменными сервисами.
- Подключение через remote entry в shell.
- Локализация и управление базовыми интерактивными состояниями.

## Что улучшить
- Вынести состояние переписок в единый store-слой.
- Усилить покрытие realtime/reconnect/failure сценариев.
- Ввести виртуализацию и оптимизацию длинных тредов.

## Техдолг
- `P0`: формализовать гарантии доставки/дедупликации/повторной отправки сообщений.
- `P1`: расширить e2e и интеграционные тесты критичных сценариев чата.
- `P2`: добавить UX/performance-метрики для длинных переписок.
