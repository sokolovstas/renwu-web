# boards

## Назначение
Микрофронтенд канбан-досок для визуального управления карточками и группами задач.

## Область ответственности
- Рендер board-экрана и интерактивные операции с карточками.
- Интеграция с доменной библиотекой `libs/board`.
- Подключение через remote entry в shell.
- Локализация и UX-состояния интерфейса доски.

## Что улучшить
- Разделить container/presentation компоненты доски.
- Добавить тесты на drag-and-drop и конкурентные изменения карточек.
- Реализовать оптимистичные обновления с rollback в ошибочных сценариях.

## Техдолг
- `P0`: формализовать инварианты перемещения карточек и обработку конфликтов.
- `P1`: расширить тестовое покрытие edge-case сценариев dnd/сортировки.
- `P2`: добавить UX/performance-метрики для тяжелых досок.
