# task

## Назначение
Микрофронтенд детальной карточки задачи с полным набором разделов редактирования и просмотра.

## Область ответственности
- Управление секциями задачи: описание, вложения, история, связи, time-log, subtasks, todo.
- Координация бизнес-операций внутри одного task-view.
- Подключение в shell и смежные приложения через remote entry.
- Локализация и единый UX-паттерн карточки.

## Что улучшить
- Вынести секции карточки в отдельные feature-модули.
- Централизовать синхронизацию изменений между секциями.
- Добавить тесты конкурентного редактирования и конфликтов версии.

## Техдолг
- `P0`: реализовать единый протокол разрешения конфликтов при параллельном редактировании.
- `P1`: унифицировать API-адаптеры и state-flow для всех секций.
- `P2`: оптимизировать тяжелые секции за счет lazy-loading и отложенных вычислений.
