# app

## Назначение
Корневой shell микрофронтенд-архитектуры Renwu: точка входа, orchestration и маршрутизация удаленных приложений.

## Область ответственности
- Инициализация глобальных провайдеров и инфраструктурных сервисов.
- Загрузка federation manifest и подключение remote-приложений.
- Глобальная навигация и интеграция разделов платформы.
- Обработка общих кросс-приложенческих сценариев ошибок.

## Что улучшить
- Вынести контракт shell-remote в отдельную спецификацию (версии, shared deps, fallback).
- Добавить e2e smoke для сценариев частичной недоступности remote.
- Ввести метрики startup/remote-load latency на уровне shell.

## Техдолг
- `P0`: стандартизировать graceful-degradation при падении или недоступности remote.
- `P1`: внедрить единый observability-слой (ошибки, web-vitals, latency интеграции).
- `P2`: ужесточить performance budget и автоматический контроль регрессий сборки.
