# projects

## Назначение
Микрофронтенд управления проектами: список, детали, команда, milestones и связанные проектные разделы.

## Область ответственности
- CRUD-сценарии проектов и проектных сущностей.
- Навигация между разделами project-домена.
- Интеграция `project.service` и shared библиотек.
- Подключение в shell через remote entry.

## Что улучшить
- Декомпозировать приложение на независимые feature-модули.
- Централизовать state-management и кэширование по вкладкам.
- Расширить контрактные тесты сервисов и ключевых user-flow.

## Техдолг
- `P0`: устранить фрагментацию бизнес-логики между секциями проекта.
- `P1`: унифицировать обработку ошибок/пустых состояний на всех экранах.
- `P2`: добавить prefetch и правила инвалидации данных на уровне вкладок.
