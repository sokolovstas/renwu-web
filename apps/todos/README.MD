# todos

## Назначение
Микрофронтенд todo-элементов с основным списком и детальным представлением.

## Область ответственности
- Отображение списка todo и детальной карточки.
- Управление пользовательскими действиями над todo-сущностями.
- Подключение к shell через remote entry.
- Локализация и отображение статусов/ошибок.

## Что улучшить
- Добавить массовые операции и undo/redo сценарии.
- Стабилизировать синхронизацию list/detail через единый state-layer.
- Расширить e2e на офлайн/онлайн и повторы отправки.

## Техдолг
- `P0`: формализовать стратегию конфликтов при изменениях из разных клиентов.
- `P1`: устранить рассинхронизацию между списком и деталями todo.
- `P2`: унифицировать UX пустых, ошибочных и подтверждающих состояний.
