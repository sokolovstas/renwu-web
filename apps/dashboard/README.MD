# dashboard

## Назначение
Микрофронтенд обзорной панели со сводной информацией и быстрыми переходами по ключевым разделам.

## Область ответственности
- Отрисовка dashboard-виджетов и агрегированных показателей.
- Интеграция с API-источниками данных для сводного экрана.
- Подключение в shell через remote entry.
- Локализация и базовые состояния загрузки/ошибок/пустоты.

## Что улучшить
- Свести запросы в единый data-layer с кэшированием и дедупликацией.
- Ввести единые правила loading/empty/error для всех виджетов.
- Добавить контрактные тесты корректности агрегированных данных.

## Техдолг
- `P0`: устранить дубли запросов между виджетами и централизовать загрузку данных.
- `P1`: стандартизировать UX-состояния для всех блоков dashboard.
- `P2`: добавить мониторинг времени рендера и загрузки виджетов.
