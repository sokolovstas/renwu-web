# settings

## Назначение
Микрофронтенд административных и системных настроек платформы.

## Область ответственности
- Управление пользователями, workflow, календарями и справочниками.
- Экранные сценарии системной конфигурации.
- Подключение в shell как remote entry.
- Локализация административных форм и параметров.

## Что улучшить
- Ввести role-based guards для экранов и действий.
- Разнести сервисы по bounded contexts (users/workflow/dictionary/system).
- Добавить интеграционные тесты для критичных административных flow.

## Техдолг
- `P0`: внедрить аудит и контроль прав на опасные операции.
- `P1`: формализовать миграции конфигураций при изменении схем данных.
- `P2`: унифицировать UX подтверждений и отката административных изменений.
