# board

## Назначение
Доменная библиотека канбан-досок: модели, сервисы и базовые UI-блоки board-сценариев.

## Область ответственности
- Модели доски, групп и карточек.
- Сервисы и бизнес-операции board-домена.
- Компоненты `card`, `group` и базовые механики сортировки.

## Что улучшить
- Разделить domain-логику и Angular presentation-слой.
- Ввести инварианты и контрактные тесты для операций перемещения карточек.
- Сократить публичный API до стабильного набора экспортов.

## Техдолг
- `P0`: формализовать правила конкурентных изменений карточек и групп.
- `P1`: унифицировать версионирование board-моделей и контрактов.
- `P2`: повысить покрытие edge-case тестами сортировки/dnd.
