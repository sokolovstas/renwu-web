# core

## Назначение
Ключевая кросс-доменная библиотека платформы: общие сервисы, модели и инфраструктурные механики.

## Область ответственности
- Общие доменные блоки (issue, search, policy, settings, state, websocket и др.).
- Кросс-приложенческие сервисы и инфраструктурные интеграции.
- Повторно используемые domain/UI компоненты общего назначения.

## Что улучшить
- Декомпозировать `core` на подбиблиотеки по bounded contexts.
- Разделить transport/state/realtime и domain/presentation уровни.
- Ужесточить контроль зависимостей через Nx tags/enforcement.

## Техдолг
- `P0`: ограничить рост `core` как "god-library" через формальный layering.
- `P1`: стабилизировать контракты сервисов состояния и websocket-потоков.
- `P2`: расширить интеграционные тесты кросс-доменных сценариев.
