# app-ui

## Назначение
Библиотека UI-каркаса приложения: layout-компоненты верхнего уровня и управление sidebar.

## Область ответственности
- Компоненты страниц (`page`, `page-with-sidebar`) и общий app-shell layout.
- Сервис состояния боковой панели и связанный UI-flow.
- Переиспользуемая инфраструктурная UI-обвязка для приложений.

## Что улучшить
- Формализовать API layout-компонентов и правила их расширения.
- Добавить визуальные regression-тесты для ключевых layout-сценариев.
- Усилить accessibility и keyboard navigation для shell-компонентов.

## Техдолг
- `P0`: зафиксировать дизайн-контракты layout-слоя и запрет ad-hoc расширений.
- `P1`: уменьшить связанность с конкретными приложениями через нейтральные интерфейсы.
- `P2`: покрыть кросс-приложенческие сценарии навигации автоматизированными тестами.
