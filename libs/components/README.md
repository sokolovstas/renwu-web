# components

## Назначение
Библиотека переиспользуемых UI-компонентов, директив, пайпов и валидаторов для всех приложений.

## Область ответственности
- Базовые и составные UI-компоненты.
- Общие директивы, пайпы и валидаторы.
- Единый визуальный и поведенческий слой интерфейса.

## Что улучшить
- Ввести строгую классификацию компонентов и lifecycle-политику API.
- Добавить визуальные тесты и sandbox-документацию (Storybook или аналог).
- Укрепить accessibility: focus management, keyboard, aria.

## Техдолг
- `P0`: убрать дублирующиеся компоненты и зафиксировать единые паттерны.
- `P1`: формализовать process breaking changes в публичном API.
- `P2`: добавить performance-тесты тяжелых UI-виджетов.
