# Tech Debt Roadmap

Сводный план по техдолгу для `apps/*` (кроме `old`) и `libs/*`.

## P0 (критично, ближайшие спринты)
- `apps/app`: стандартизировать graceful-degradation при недоступности remote.
- `apps/boards`: формализовать инварианты dnd и конфликтов перемещения карточек.
- `apps/dashboard`: централизовать data-layer и убрать дубли запросов виджетов.
- `apps/documents`: ввести единый протокол ошибок/таймаутов для операций с документами.
- `apps/messenger`: зафиксировать гарантии доставки и дедупликации сообщений.
- `apps/profile`: усилить контроль прав и валидацию критичных изменений профиля.
- `apps/projects`: устранить фрагментацию бизнес-логики между секциями.
- `apps/settings`: внедрить аудит и контроль прав опасных административных действий.
- `apps/task`: реализовать единый протокол разрешения конфликтов редактирования.
- `apps/tasks`: синхронизировать бизнес-логику между list/tree/timeline.
- `apps/timeline`: закрыть визуальные регрессии regression-тестами.
- `apps/todos`: формализовать стратегию конфликтов при изменениях из разных клиентов.
- `libs/app-ui`: зафиксировать дизайн-контракты layout-слоя.
- `libs/board`: формализовать конкурентные изменения карточек/групп.
- `libs/components`: убрать дублирующиеся компоненты и единообразить паттерны.
- `libs/core`: ограничить разрастание `core` через формальный layering.
- `libs/mentions`: стабилизировать caret-логику и навигацию подсказок.
- `libs/messaging`: стандартизировать идемпотентность и дедупликацию событий.
- `libs/utils`: остановить рост дубликатов и legacy helper-функций.

## P1 (важно, после закрытия P0)
- Декомпозиция крупных модулей (`apps/projects`, `apps/task`, `libs/core`) на feature/contexts.
- Централизация state-management в `apps/tasks`, `apps/task`, `apps/messenger`.
- Усиление интеграционных и e2e тестов для критичных пользовательских flow.
- Формализация контрактов API DTO -> UI model в `apps/profile` и смежных доменах.
- Унификация миграций и схем конфигураций в `apps/settings`.

## P2 (улучшения устойчивости и DX)
- Performance budget и мониторинг рендера для `apps/timeline`, `apps/dashboard`, `apps/messenger`.
- Визуальные regression-тесты для `libs/app-ui` и `libs/components`.
- Стандартизация UX-состояний (loading/empty/error/confirm) во всех приложениях.
- Оптимизация публичных API библиотек и проверка tree-shaking (`libs/components`, `libs/utils`).

## Рекомендуемый порядок внедрения
1. Закрыть P0 по отказоустойчивости, конфликтам редактирования и консистентности данных.
2. Параллельно усилить тестовую стратегию (integration + e2e + regression).
3. Выполнить P1 декомпозицию модулей и стабилизацию архитектурных контрактов.
4. Закрыть P2 задачи производительности, UX-стандартизации и DX-оптимизации.
